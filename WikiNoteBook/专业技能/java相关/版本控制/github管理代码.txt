====== github管理代码 ======
Created Friday 13 January 2017

===== git入门之github初探 =====
序言
        自从进入公司以后，开发的时候就使用到了gitflow的开发流程，git的话，虽说是接触到了，但是一直使用的是sourcetree进行开发，那种图形化的界面虽然可以让我们开发人员的学习曲线变得很平缓，但是当我面对github的时候，对于一些细节我还是不太了解，最近这两天想把自己的代码使用github管理起来，毕竟现在不是好多人都说看一个程序员的资历从github的主页就可以看得出来！但是当自己动手的时候，才发现公司自建的代码仓库隐藏了太多的细节，导致小生我折腾了半天，那么就对自己这个折腾的过程来进行各总结吧！
大致流程
        大致流程如下：
	首先，当然申请一个github的账号。
	新建一个仓库，这个的话，在很显眼的位置，也就不说啥了。
安装git的客户端，我目前使用的是：Git-2.5.2.2-64-bit
首先选定一个目录，这就是你要存放项目的目录，在这个目录下使用 git init ，当然是在命令行下进行的，也可以在图像化界面上操纵，现在的客户端基本都提供这两种方式的。
这个时候，我们可以发现我们本地的目录的地方已经存在一个.git目录，但是你有可能看不到，那是因为那是隐藏文件夹啊。至于如何看到它，那就去找下了。
现在，我们就要连接github服务器了，但是你连接它总要有个密匙凭证，那就在你本地生成一个，生成的办法就是：ssh-keygen -t rsa -C "943065465@qq.com" 当然，系统会要求你输入用户名密码之类的，一路回车即可。
这个时候就要去看看我们的ssh key文件是否成功生成了。那你想想一定在我们的系统的管理员目录下面了，这个时候，打开windows powershell，输入 cd ~，进入系统管理员主目录，然后cd .ssh,可以看到有几个文件，id_rsa.pub就是我们需要的，粘贴里面的内容即可。
进入https://github.com/settings/keys，当然你一定要提前登录，直接配置就好，期间可能会要求你输入密码。
上面的步骤做完，我们就大功告成了，检测一下，ssh -T git@github.com 恩，就是的，你可以看到已经成功连接。
接下来就要上传本地仓库上去了，这个时候你需要先设置一下自己的名字和邮箱，因为每次提交都会把相应的信息提交上去。 就是这麽个设置法：
git config --global user.name "your name"
$ git config --global user.email "your_email@youremail.com"
11.下面要做的就是添加远程地址了，这是和github服务器真正开始连接的第一步，
git remote add origin git@github.com:yourName/yourRepo.git
连接上，你会发现.git目录下面出现了一个config文件，里面也会相应的出现远程服务器的地址，当然啦，你也可以直接修改config文件改变远程服务器地址。
        上面步骤我们做一个总结，这就是连接github服务器的一些操纵，其中这些操作除了第十一步是每次都要操作的，别的都是只需要配置一次。
git基本操作
        上面的话其实我们已经谈到，配置已经完成，下面就以最简单的例子，上传一个文件做一个测试。         首先新建hello.txt,将其纳入我们的代码控制中去。
git add hello.txt
之后提交到本地，命令如下：
git commit -m "first commit"
最后，先和远端同步，再将远程代码拉下来：
git pull origin master
git push origin master
到了这里的话，就大功告成了。我们的项目已经成功的使用github管理起来了，当然，要想熟练的使用git，我们的路才刚刚开始。
